DB Vault — Setup Guide

Install the access-broker package, wire up phpMyAdmin, and (optionally) add mutual-TLS device authentication.

What the package provides vs. what you set up. The composer package ships all the logic (RBAC, 2FA, provisioning, cert issuance, the SPA). The steps below configure the environment it runs in — a MySQL client to land in (phpMyAdmin), and optionally a reverse proxy for mTLS. Only phpMyAdmin is required for the launch flow to work.

1 · Install the package

Require via composer

composer require ak-codehub/db-vault

Configure the vault's own database

The vault stores its tables on a dedicated connection, isolated from your app DB. In .env:

DBVAULT_DB_DATABASE=dbvault
DBVAULT_DB_HOST=127.0.0.1
DBVAULT_DB_USERNAME=root
DBVAULT_DB_PASSWORD=secret

Build the SPA, then install

Build the front-end before installing (install publishes the built assets):

npm install && npm run build          # in the package dir
php artisan db-vault:install            # migrates, seeds roles, creates first admin

Serve & open

php artisan serve   #  → http://127.0.0.1:8000/vault

2 · phpMyAdmin (native) required

When a user launches a session, DB Vault mints a one-time token and sends them to phpMyAdmin, which trades the token for the temporary MySQL credential and logs them in automatically — no password ever shown. phpMyAdmin is served natively under your app's own URL at {DBVAULT_PATH}/pma (e.g. https://your-app.example.com/vault/pma) — same origin, same TLS cert, no separate port and no separate vhost. Two artisan commands install and wire it; phpMyAdmin itself is downloaded onto your host (not bundled in the package).

1 · Download phpMyAdmin

Fetches a pinned, SHA-256-verified release into storage/app/dbvault-pma and overlays the signon config:

php artisan dbvault:install-pma

2 · Generate the web-server block

Auto-detects nginx vs Apache and the PHP-FPM socket, and prints a fully-resolved block (real path, your {vault-path}/pma prefix, the signon secret, and PmaAbsoluteUri from APP_URL). Paste it inside your app's existing HTTPS server {} / <VirtualHost> block, then reload:

php artisan dbvault:pma-vhost                # or --server=apache

3 · Set the secret in .env

Must match the value carried by the generated block, then php artisan config:clear. Leave DBVAULT_PMA_SIGNON_URL unset — launch defaults to {vault-path}/pma/signon.php:

DBVAULT_SIGNON_SECRET=<a long random string>
If "Open phpMyAdmin" loops back to its own UI: the exchange returned 403 because DBVAULT_SIGNON_SECRET is empty or doesn't match the vhost's value. This is the most common failure. 404 at /vault/pma: phpMyAdmin isn't installed (re-run dbvault:install-pma — storage/app/dbvault-pma is git-ignored and cleared by some deploys) or the vhost path is wrong (re-run dbvault:pma-vhost).

3 · Certificate Authority optional

Only needed for mTLS. The package can issue client certificates itself once it has a CA. Generate a local one with the built-in artisan command (no script needed):

php artisan dbvault:make-ca
# writes storage/app/dbvault-ca/{ca.crt,ca.key} — point .env at them:
#   DBVAULT_CA_CERT=.../ca.crt
#   DBVAULT_CA_KEY=.../ca.key

Then issue per-user certs from Admin → Devices → Issue certificate in the panel.

4 · nginx mTLS reverse proxy optional

Terminates HTTPS, verifies client certs against your CA, and forwards the result to the app.

chmod +x setup-nginx-mtls.sh
sudo CA_CERT=/path/to/storage/app/dbvault-ca/ca.crt \
     LISTEN_PORT=8443 APP_UPSTREAM=127.0.0.1:8000 \
     ./setup-nginx-mtls.sh

4b · apache mTLS reverse proxy optional

Use instead of nginx if you run apache.

chmod +x setup-apache-mtls.sh
sudo CA_CERT=/path/to/ca.crt ./setup-apache-mtls.sh

5 · Enable mTLS enforcement optional

Once a proxy is in front, turn on the gate in the vault host .env:

DBVAULT_MIDDLEWARE="vault.mtls,web"
DBVAULT_MTLS_REQUIRE_ENROLLED_DEVICE=true
Do not enable this without a proxy in front. vault.mtls trusts cert headers a reverse proxy injects. On a bare php artisan serve every request will 403.
Also trust the proxy or the SPA renders blank (assets load as http). In bootstrap/app.php → withMiddleware(): 
$middleware->trustProxies(at: '*', headers: Request::HEADER_X_FORWARDED_FOR
    | Request::HEADER_X_FORWARDED_HOST | Request::HEADER_X_FORWARDED_PORT
    | Request::HEADER_X_FORWARDED_PROTO);
Bootstrapping: first cert without panel access

Enrolling happens inside the gated panel — so issue the first admin cert out-of-band via tinker, then log in and manage the rest from the UI.

$u = DbVault\Models\User::where('email','admin@example.com')->first();
$r = app(DbVault\Services\CertificateAuthorityService::class)->issueForUser($u,'bootstrap');
file_put_contents('admin.p12', $r['p12']);   // install; password in $r['password']
DbVault\Models\Device::create([
  'user_id'=>$u->id,'cert_dn'=>$r['dn'],'cert_fingerprint'=>$r['fingerprint'],
  'label'=>'bootstrap','enrolled_at'=>now(),
]);
